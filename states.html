The 3 States of Data

To be able to properly protect data, the state of the data is very important. These three states are Data at Rest, Data in Transit, and Data in Use. Data can very quickly and frequently change from state to state, or it can stay the same state for its entire lifecycle.

Source: The New York Times

Data at Rest

     Data is considered at rest when it is not currently being accessed or transferred. Another way to describe data at rest is when data is in storage. An example of this is a file inside of a hard drive or data base which is not being accessed or modified.

Data in Transit

     Data in transit refers to data when it is being moved from one place to another. This is usually between locations on a single computer or between computers, such as the cloud. Data moving within the RAM of a computer, files getting dragged from one folder to another, and emails are examples of data in transit.

Source: MakeUseOf

Source: How-To Geek

Data in Use

     When any type of data is being processed or changed, it is in use. This includes updating, accessing, and creating data. When a file is created or accessed in a system, the data is considered in use and is at its most vulnerable to attacks. This is because the data is directly accessible by one or more users.
